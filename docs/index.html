tasksAgent Documentation

This repository currently hosts one Ploinky agent: tasksAgent.

The agent is integrated in Ploinky and is used by Explorer flows to manage repository backlog files through MCP tools.

All task tools execute through tasksAgent/tools/tasks_tool.sh, which dispatches to tasks_tool.mjs.

Operational Role

tasksAgent mediates between Explorer task interactions and backlog-file execution through MCP tools that cover configuration, listing, retrieval, mutation, history, and reorder operations.

The agent enforces repository path boundaries and file-extension rules before any mutation, and Explorer integration is delivered through tasksAgent/IDE-plugins/tasks-tool-button/ as the UI entry channel.

Repository Scope

The repository can host multiple Ploinky agents over time. This documentation set targets the current tasksAgent scope and its role as the operational bridge between Explorer and backlog management.

Validation rule: run the tasks agent tests under tasksAgent/tests for code changes, together with documentation and specification updates.
