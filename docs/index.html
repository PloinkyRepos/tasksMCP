tasksAgent Documentation

This repository currently hosts one Ploinky agent: tasksAgent.

The agent is integrated in Ploinky and is used by Explorer flows to manage repository backlog files through MCP tools.

All task tools execute through tasksAgent/tools/tasks_tool.sh, which dispatches to tasks_tool.mjs.

What this agent does

  • Acts as intermediary between Explorer task UI and backlog file operations.
  • Exposes controlled MCP tools for config, list, get, create, update, delete, history, and reorder flows.
  • Enforces path scope rules so backlog and history files stay inside the selected repository root.
  • Supports Explorer integration via tasksAgent/IDE-plugins/tasks-tool-button/.
